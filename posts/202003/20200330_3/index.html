pyenv 環境下で git submodule が使えなくなったときのメモ

Posted at — Mar 30, 2020

git submodule コマンドを使おうとしたら pyenv のパスを参照して gettext.sh が無いとエラーが出た．

/usr/lib/git-core/git-submodule: 6: /home/USERNAME/.pyenv/shims/gettext.sh: [[: not found

解決方法

ここ を参照．

The actual problem is in git-core. It seems it's trying to search gettext.sh from $PATH then source it.

とのこと．

$ export GIT_INTERNAL_GETTEXT_TEST_FALLBACKS=1

で解決できる
