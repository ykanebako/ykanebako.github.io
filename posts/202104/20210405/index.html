毎日書くことが大切

Posted at — Apr 5, 2021

最近，人生で初めて英語でフルペーパーの論文を書いてサブミットした．まずは打席に立ってバットを振れたことが素直に嬉しい．次はヒット (accept) を打ちたい．

論文執筆は簡単ではなく，スケジュール的にもギリギリでほとんど推敲する時間が無かったことが反省点．理想としては締切の2週間前，可能なら1ヶ月前に書き上げて，そこから何度も推敲を重ねたいところ．余裕を持って書き上げるには，できる研究者の論文生産術 でも書かれているが，毎日執筆時間を確保することが重要らしい．企業勤めで研究部門でない自分でも論文を出し続ける（そうじゃなくても日々のドキュメンテーションを効率良く行う）ために，次のことを意識したい．

論文の読み方・まとめ方

論文を読んだら以下については真面目にメモする．

  • 提案手法の概要
  • 提案手法のモチベーション・着想
  • 実験結果，または不足していそうな実験 (Negativeな結果)

以前は 落合先生のフォーマット で論文をまとめていたが，画像まで貼るのが面倒でやめてしまっていた．その後，落合先生のフォーマットを独自に改良してメモしていたが，画像を貼らない代わりに項目を増やしてしまったため，結局続かなかった．なので，まずは最低限上記の3点を抑えるように読み，メモしたい．

実験結果を丁寧に残す

以下について必ずメモする

  • 実験の目的，仮説
  • 検証条件
  • 結果，考察，次にやること

普通誰もがやっていることだけど，しっかり明文化することを意識したい．あと最近，実験結果の管理に MLflow と Hydra を導入したらいい感じだったので引き続き使っていきたい．

ツール

論文管理は Paperpile，ソースコード管理は Github でやるとして，ノートテイキングはとりあえず Google Docs でやる．ある程度結果とかがまとまってきたタイミングで Overleaf とかでしっかり文書に残すと良さそう．

まずは一ヶ月やってみる．
